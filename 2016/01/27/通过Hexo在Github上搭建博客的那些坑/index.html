通过Hexo在Github上搭建博客的那些坑

发表于 2016-01-27   |     |   阅读次数

一、作者叨叨叨

这篇文章的目的是为了记录在搭建Github博客时出现的坑。很遗憾，本人在根据网络教程搭建博客时，几乎每一步都出现了问题，so sad ~

所以记录一下正确运行结果，错误运行结果，以及解决方案，供参考。如果你已搭建成功，不需要看此文章。

  • 时间：2016年1月27日。
  • 环境：MAC OS X El Captian 10.11
  • Node版本：4.2.3

三、趟过的那些坑

  • 坑1：-bash: npm: command not found

    假设，node环境你已安装完成，但是执行npm命令时报错。（别问我为什么，因为我之前就安装成功了，不成功网上也有好多解决方案）

    • npm错误结果，如下图：

    • 解决方案：

      bash出错，一般是环境变量没有配置好。试用了网上所有方法都不管用。所以，只能重新安装了。

      npm是在安装node时顺带安装的。重新命令行安装node环境，会提示node已安装。所以，我最后的解决办法是，下载 node-v4.2.3.pkg 安装包，手动安装，就可以了。

      安装完成会提示node安装成功，npm安装成功。这时，npm命令就可以使用啦~~ 如下图：

  • 坑2：npm install hexo -g 报错

    • 错误如下：

    • 错误原因：

      安装目录出错。npm安装是在当前目录下安装，最后安装的位置是/usr/local/bin 。虽然上面的命令是在全局下安装，但是执行命令的目录不能是node的安装目录，需要换到根目录或者其他目录下即可。

  • 坑3： hexo generate 报错

    • 错误如下：

      ps： hexo g 等同于 hexo generate，命令行会自动补全命令。下同。

    • 解决方案：

      我自己建了一个目录叫blog，在此目录下存放我的博客内容。hexo init 成功之后，generate报错。一开始还在网上各种查找方法尝试，最后乖乖看提示，试用了下提示的try running，结果就好了。

      所以，一定要看错误提示。执行命令：npm install hexo –save 即可。

    • 正确运行结果：

      也可以是这样的：

      ps: 
如上图红线标注，一定要出现 hello-world文件才算创建成功。
如果也出现了粉红色字体创建，但是没有hello-world文件创建，那就是没有成功。
需要要你调用 hexo clean 方法先clean，再重新执行generate创建一次。

  • 坑4： hexo server 报错

    • 错误如下：

      结果中没有报错，也没有返回正确结果，只呈现了hexo所包含的命令行。这样是不正确的，我们先看下正确的结果。

* **正确运行结果：**

    http://localhost:4000/  可以访问成功。


* **解决方案：**

    只要上一步骤 hexo generate 正确了，这步就不会错。所以，上一步 generate 一定要出现hello-world 文档。所以sever出错，要去检查上一步。

  • 坑5： hexo deploy #进行部署 报错

    • 错误如下：

      在config配置文件中，最后type类型填写 git 或者 github都会报错。

      ps：config文件中，deploy: type: 等标签后面一定要有『空格』，否则命令不执行，不报错。

    • 解决方案：

      需要执行命令行：npm install hexo-deployer-git –save，因为在Hexo 3.0 后server被单独出来了，需要先安装server。如下图：
